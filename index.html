BESLUTNINGSGUIDE

Hvordan velge riktig plattform

Hostingvalget avhenger av hva du skal deploye og hvor mye trafikk du forventer. Her er beslutningstreet som fungerer for 95 prosent av norske prosjekter.

Hvis du bygger en statisk side eller Next.js

Start med Vercel gratistier. Hvis du overstiger 100 GB båndbredde per måned, vurder Cloudflare Pages med ubegrenset båndbredde. Begge deployer fra GitHub i 30 sekunder.

Hvis du bygger en backend eller API

For små og variable belastninger: Railway med usage-basert prising. For 24/7 global edge: Fly.io. For maksimal kostnadskontroll: Hetzner VPS.

Hvis du trenger full kontroll og europeisk datalokasjon

Bruk Hetzner i Nürnberg eller Falkenstein, eller DigitalOcean i Frankfurt. Begge gir deg ren Ubuntu-VPS med GDPR-trygg lagring, fra 45-75 kr per måned.

Hvis du er i enterprise eller sertifisert sektor

Velg AWS eller Azure. Dyrere og mer komplekst, men du får ISO 27001, SOC 2 og tusenvis av tjenester i samme konto.

Vanligste feilvalg i 2026

  • Vercel for Python-backender. Bruker du cron-jobber eller langlevd prosesser, velg Railway eller Fly.io.
  • AWS for småsider. Overkill og lett å få overraskelsesregning. Start med Vercel eller Hetzner.
  • Hetzner uten linux-erfaring. Du får ren VPS uten kontrollpanel. Bruk Railway eller DigitalOcean App Platform hvis du ikke kjenner SSH.
  • Netlify uten spend cap. Båndbreddsoverkjøringer koster 55 usd per 100 GB, langt dyrere enn Vercel.

Viktige observasjoner

OFTE STILTE SPØRSMÅL

12 svar på det folk lurer på

De vanligste spørsmålene fra norske gründere og utviklere som skal velge hosting. Klikk på hvert for svar.

Hva er forskjellen på web-hosting og en hosting-plattform?
Klassisk web-hosting (shared hosting, VPS) gir deg en server der du selv må installere alt. Moderne hosting-plattformer som Vercel eller Railway håndterer build, deploy, SSL, skalering og overvåkning automatisk. Du pusher til Git og alt fungerer. For 2026-prosjekter er plattformer nesten alltid riktig valg med mindre du har spesielle krav.
Kan jeg hoste en nettside gratis i 2026?
Ja, kommersielt. Vercel, Netlify og Cloudflare Pages har alle gratistier som dekker SMB-prosjekter. Cloudflare er mest generøs med ubegrenset båndbredde. Du betaler bare for domenet (149 kr/år for .no). Gratistiers varer i evig tid så lenge du ikke overstiger limits.
Hvorfor velge Vercel hvis Cloudflare Pages er billigere?
Vercel er laget av Next.js-teamet. Nye Next.js-funksjoner fungerer først og best der. ISR (Incremental Static Regeneration), Server Components, Image Optimization. Cloudflare Pages støtter Next.js men har alltid noen måneders forsinkelse på nye funksjoner. For seriøse Next.js-prosjekter er Vercel verdt ekstra-kostnaden.
Er GDPR et problem med amerikanske plattformer?
Det er komplisert etter Schrems II-dommen (2020). I 2026 bruker de fleste norske nettsteder fortsatt Vercel, Netlify og Cloudflare med DPA-avtaler og Standard Contractual Clauses. For maksimal compliance: velg Hetzner i Tyskland eller DigitalOcean Frankfurt-regionen. For offentlig sektor: krev ofte datalagring i Norge eller EØS.
Bør jeg bruke WordPress eller Next.js i 2026?
WordPress for innholdstunge sider (blogg, nettavis, bedriftsside) hvor redaktører må redigere uten utviklere. Next.js for alt annet: nettbutikker, apper, SaaS, markedsføringssider med tilpasset design. Next.js er raskere, tryggere og billigere å hoste, men krever utvikler for endringer. AI-verktøy som Claude kan redusere denne terskelen betydelig.
Hva koster det å hoste en nettbutikk i Norge?
Shopify: 290-3000 kr/mnd inkludert hosting. Custom nettbutikk på Vercel: 200-400 kr/mnd. WooCommerce på managed hosting: 250-1500 kr/mnd. Legg til Supabase eller PostgreSQL for 0-200 kr/mnd. Total: 200-3500 kr/mnd avhengig av plattform.
Hvor lang tid tar det å bytte hosting?
Statisk side eller Next.js: 30 minutter (git push til ny plattform). WordPress-side: 4-20 timer avhengig av størrelse. Full VPS-migrering: 1-5 dager. Viktigst: test alt på ny server før du endrer DNS. Nedetid kan unngås helt med riktig planlegging.
Hvorfor er shared hosting (One.com, Domeneshop) tregt?
Du deler server med hundretusenvis av andre nettsteder. CPU, RAM og diskkapasitet er begrenset. Når en nabo-nettside får trafikktopp, lider din side. Moderne plattformer som Vercel eller Cloudflare bruker isolerte containere og CDN-edge slik at naboer ikke påvirker deg.
Må jeg ha et .no-domene som norsk bedrift?
Nei, men det gir tillit. Norid (som administrerer .no) krever norsk org.nr for .no-domener. .no-domener er SEO-fordel for norsk målgruppe og signaliserer seriøsitet. .com eller .io fungerer fint hvis du har internasjonal ambisjon. Mange norske bedrifter har både (eks. bedrift.no og bedrift.com).
Trenger jeg CDN hvis jeg bruker Vercel?
Nei, Vercel inkluderer eget edge-nettverk. Det samme gjelder Netlify og Cloudflare Pages. CDN er først relevant hvis du bruker klassisk VPS (Hetzner, DigitalOcean) eller shared hosting uten innebygd edge. Da legger du Cloudflare gratis foran.
Hva skjer når jeg overgår gratistiers?
Varierer per plattform. Vercel blokkerer gratistier-overforbruk — siden går i wait. Cloudflare Pages fortsetter å serve båndbredde gratis men limiter Workers-requests. Netlify sender deg faktura uten varsel hvis du ikke har satt spend cap. Sett alltid spending limits før du lanserer noe med trafikkpotensial.
Hvilken plattform er best for norske bedrifter i 2026?
For 80 prosent: Vercel for frontend og Railway for backend. For kostnadsfokuserte: Cloudflare Pages og Hetzner. For WordPress: Kinsta eller WP Engine. For enterprise med compliance-krav: AWS Frankfurt eller Azure North Europe. Ingen universell vinner — avhenger av prosjekt og team.
ORDLISTE

Terminologi for hosting

Hosting-bransjen er full av akronymer. Her er 20 av de viktigste termene på norsk.

CDN

Content Delivery Network. Nettverk av servere globalt som cacher og leverer statiske filer. Reduserer TTFB og båndbredde fra opprinnelsesserver.

Edge

Servere nær sluttbrukeren. Cloudflare har 330+ edge-lokasjoner. Lavere latens enn sentraliserte datasentre.

TTFB

Time To First Byte. Tid fra forespørsel til første svar-byte. Under 200 ms er bra, under 100 ms er utmerket. Core Web Vitals måler dette.

VPS

Virtual Private Server. Virtualisert server der du får root-tilgang. Hetzner og DigitalOcean er største leverandører av ren VPS i 2026.

Serverless

Kode som kjører kun når det trengs. Du betaler per invokering, ikke per måned. Vercel Functions og AWS Lambda er eksempler.

Cold start

Forsinkelse når serverless-funksjon starter etter inaktivitet. 100-400 ms typisk. Cloudflare Workers har ingen cold start pga V8 isolates.

Egress

Data som sendes ut fra server til klient. AWS tar 0,09 usd/GB i egress. Cloudflare R2 har null egress. Kan være stor skjult kostnad.

DNS

Domain Name System. Oversetter domenenavn til IP-adresser. Cloudflare DNS er raskest og gratis. Propagering tar 0-48 timer globalt.

SSL og TLS

Kryptering mellom klient og server. TLS er moderne versjonen. Let's Encrypt gir gratis sertifikater med automatisk fornyelse.

WAF

Web Application Firewall. Blokkerer OWASP Top 10-angrep (SQL injection, XSS, etc.) før de når din app. Cloudflare WAF inkludert fra Pro-plan.

DDoS

Distributed Denial of Service. Angrep som overbelaster server. Cloudflare og Vercel har innebygd DDoS-beskyttelse på alle planer.

Uptime

Tid serveren er tilgjengelig. 99,9 prosent = 8,76 timer nedetid per år. 99,99 prosent = 52 minutter. Enterprise krever 99,99+ prosent.

SLA

Service Level Agreement. Kontrakt som garanterer uptime. AWS SLA: 99,99 prosent. Vercel SLA: 99,9 prosent på Pro. Brudd gir kreditt, ikke refusjon.

SSG

Static Site Generation. Bygger HTML på forhånd. Raskest mulig leveranse. Next.js, Astro og Hugo støtter dette.

SSR

Server-Side Rendering. HTML genereres per forespørsel. Nødvendig for dynamisk innhold som brukerprofiler. Tregere enn SSG.

ISR

Incremental Static Regeneration. Next.js-funksjon som regenererer statisk HTML etter tidsperiode. Balanse mellom SSG og SSR.

Git push deploy

Deployment som utløses når du pusher til Git. Vercel, Netlify, Cloudflare Pages og Railway støtter dette fra GitHub, GitLab og Bitbucket.

Preview deploy

Unik URL for hver pull request. Tester endringer før merge. Standard på Vercel, Netlify og Cloudflare Pages.

Headless

Arkitektur der backend (API) er skilt fra frontend. Samme API serverer web, app, POS. Brukes mye i moderne commerce.

Container

Isolert prosess med alle dependencies. Docker er standard. Railway, Fly.io og AWS ECS kjører containere. Gir konsistent deployment.
