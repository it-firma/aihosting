OPPDATERT 24. APRIL 20262100+ ORDLESETID 9 MIN

Hva er DigitalOcean?

DigitalOcean er en amerikansk skyleverandør lansert i 2011, posisjonert som "cloud for developers" — enklere enn AWS men med lignende kjernefunksjoner. Flaggskipet er Droplets (klassiske VPS-er), men de har også App Platform (hosted apps som Heroku), Managed Databases, Spaces (S3-kompatibel storage) og Kubernetes.

STABIL STANDARD

DigitalOcean er ikke raskest eller billigst, men pålitelig og velkjent. 15+ års erfaring, tydelig dokumentasjon, gode priser. En "trygg middelvei" for norske bedrifter.

DigitalOcean-produkter

Droplets (VPS)

Klassiske virtuelle maskiner. Du får ren Ubuntu/Debian/CentOS med root-tilgang. Du installerer alt selv. Priser:

  • Basic 1 vCPU, 1 GB, 25 GB SSD: 45 kr/mnd (4 usd)
  • Basic 1 vCPU, 2 GB, 50 GB SSD: 90 kr/mnd (8 usd)
  • Basic 2 vCPU, 4 GB, 80 GB SSD: 180 kr/mnd (16 usd)
  • CPU-Optimized: Fra 360 kr/mnd (dedicated vCPU)
  • Memory-Optimized: Fra 540 kr/mnd (for databaser)

App Platform (PaaS)

Heroku-likt grensesnitt: push til GitHub, DigitalOcean bygger og deployer. Inkluderer SSL, autoscaling, custom domains. Fra 60 kr/mnd for basic apps.

Managed Databases

PostgreSQL, MySQL, MongoDB, Redis. Fra 150 kr/mnd for minste instans. Automatiske backups, read replicas, connection pooling.

Spaces (Object Storage)

S3-kompatibel. 250 GB for 50 kr/mnd. CDN inkludert. Billigere enn AWS S3 men dyrere enn Cloudflare R2 (som har null egress).

Kubernetes (DOKS)

Managed Kubernetes-cluster. Control plane gratis, du betaler for worker nodes (ca 180 kr/mnd per node for Basic).

Droplet-deployment

# 1. Opprett droplet via dashboard eller CLI (doctl)
doctl compute droplet create web-01 \
  --region fra1 \
  --image ubuntu-22-04-x64 \
  --size s-2vcpu-4gb \
  --ssh-keys $(doctl compute ssh-key list --no-header --format ID | head -1)

# 2. SSH inn
ssh root@YOUR_DROPLET_IP

# 3. Installer stack (eksempel: Node.js + Caddy)
apt update && apt install -y nodejs npm caddy
systemctl enable caddy

# 4. Deploy app
cd /var/www && git clone https://github.com/dittprosjekt/app.git
cd app && npm install && npm run build

# 5. Caddyfile for automatisk HTTPS
cat > /etc/caddy/Caddyfile << EOF
dittdomene.no {
    reverse_proxy localhost:3000
}
EOF
systemctl restart caddy

Styrker og svakheter

Styrker

  • Beste dokumentasjon i bransjen. DigitalOcean Community har tutorials for nesten alt.
  • Forutsigbar prising. Månedspris, ingen bandwidth-overraskelser (500 GB inkludert per Droplet).
  • Enkelt dashboard. Mye enklere enn AWS.
  • EU-datasenter. Frankfurt er GDPR-trygg for norsk bruk.
  • 200 usd gratis kreditt. Nok til å teste i 2 måneder.
  • Marketplace med 1-click apps. WordPress, LAMP stack, Mattermost — forhåndskonfigurerte droplets.

Svakheter

  • Ingen ekte gratis tier. Kredittet er prøveperiode.
  • Tregere enn Hetzner. Hetzner CX22 gir mer for mindre penger.
  • Færre regioner enn AWS. Men fler enn Hetzner.
  • Managed Postgres dyrere. Vs Fly.io Postgres.
  • App Platform ikke like polert. Som Vercel eller Railway.

DigitalOcean vs Hetzner for norske bedrifter

Direkte sammenligning for 4 GB RAM VPS:

  • Hetzner CPX21: 2 vCPU, 4 GB, 80 GB SSD — 90 kr/mnd
  • DigitalOcean Basic: 2 vCPU, 4 GB, 80 GB SSD — 180 kr/mnd

Hetzner er halv pris. Men DigitalOcean har bedre dokumentasjon, enklere dashboard, og Marketplace-apper. For nybegynnere er DigitalOcean trolig verdt prisforskjellen.

Når bør du velge DigitalOcean

Riktig hvis:

  • Du vil ha klassisk VPS med root-tilgang
  • Du er mellom-erfaren utvikler (bedre enn AWS, ikke like bar-bones som Hetzner)
  • Du trenger App Platform for enkle deploys uten å håndtere server
  • Du vil ha EU-datalokasjon (Frankfurt)
  • Du trenger managed Kubernetes

Ikke riktig hvis:

  • Pris er hovedprioritet (Hetzner bedre)
  • Du vil ha moderne DX og git-push-deploy (Railway, Vercel)
  • Enterprise-compliance (AWS, Azure)
