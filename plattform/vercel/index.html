Amerikansk plattform som eier Next.js. Best DX, raskest tilgang til nye funksjoner, men høyere pris enn alternativene. Her er full gjennomgang.

Hva er Vercel?

Vercel er en amerikansk hosting-plattform lansert i 2015 (opprinnelig som ZEIT). Selskapet er hjemmet til Next.js, det mest populære React-rammeverket i 2026. Dette gir Vercel en unik posisjon: nye Next.js-funksjoner fungerer først og best der, og plattformen er optimalisert for moderne JavaScript-rammeverk fra grunnen av.

STANDARDEN FOR NEXT.JS I 2026

Hvis du bygger med Next.js er Vercel riktig valg for 95 prosent av tilfeller. ISR, Server Components, Image Optimization, Edge Functions — alt fungerer sømløst. For andre rammeverk finnes det billigere alternativer.

Vercel-planene 2026

Hobby — gratis

For personlige prosjekter og hobby. Inkluderer: 100 GB båndbredde, 1 million Edge Function-invokeringer, automatisk HTTPS, preview deploys, analytics. Merk: ikke-kommersielt bruk — Vercel tolker dette fleksibelt, men formelt sett.

Pro — fra 200 kr/mnd per bruker

Kommersiell bruk. 1 TB båndbredde inkludert, ubegrensede team-medlemmer (i 2026-planen), spend limits, 30-dagers analytics, passordbeskyttelse. Ekstra båndbredde: 150 kr per 100 GB. Dette er standarden for norske SMB-er.

Enterprise — fra 20000 kr/mnd

SSO, dedicated support, custom SLA (99,99 prosent), HIPAA og SOC 2. For større organisasjoner eller apper med compliance-krav.

Hva gjør Vercel unikt

Fluid Compute

Vercel introduserte Fluid Compute i 2025-2026, som holder serverless-funksjoner warme og lar dem håndtere flere samtidige requester. Dette reduserte cold start-problemet betydelig — nå p99 latency forbedret med ~25 prosent sammenlignet med 2025-benchmarks.

Image Optimization

Neste/image-komponenten i Next.js leverer automatisk WebP/AVIF, responsive bilder og lazy loading via Vercel's bilde-pipeline. Andre plattformer støtter dette, men Vercels versjon er mest polert.

Preview Deploys

Hver pull request får unik URL. Team kan teste endringer før merge. Dette er blitt standard på alle moderne plattformer, men Vercel har den mest polerte implementasjonen med kommentarer direkte i preview.

Instant Rollback

Én knapp for å rulle tilbake til forrige deployment. Kritisk når produksjon brekker. Fungerer på Pro og over.

Slik deployer du på Vercel

Fra GitHub (anbefalt)

# 1. Push kode til GitHub
git push origin main

# 2. Gå til vercel.com og importer repoet
#    Vercel oppdager automatisk rammeverk (Next.js, Astro, Vite, etc.)

# 3. Sett environment variables hvis nødvendig
DATABASE_URL=postgres://...
STRIPE_KEY=sk_live_...

# 4. Klikk Deploy. Første build tar 60-120 sekunder.
# 5. Produksjons-URL: https://ditt-prosjekt.vercel.app

# 6. Legg til eget domene under Settings → Domains
#    DNS-oppsett vises automatisk

Fra CLI

npm install -g vercel
vercel login
vercel # preview deploy
vercel --prod # produksjon

Styrker og svakheter

Styrker

  • Beste Next.js-opplevelse. Nye funksjoner fungerer først her.
  • Rask og polert DX. Dashboard, CLI, GitHub-integrasjon — alt er godt designet.
  • Fluid Compute. Løste cold start-problemet som plagde serverless tidligere.
  • Global edge. Ikke like utbredt som Cloudflare, men raskt nok for 99 prosent av brukere.
  • Innebygde tjenester. Vercel Postgres, Blob Storage, KV, Edge Config — integrert med hovedplattformen.
  • Preview deploys for alle. Gratis på alle planer.

Svakheter

  • Pris skalerer raskt. Pro fra 200 kr/mnd per bruker + 150 kr/100 GB båndbredde. Ved høy trafikk blir det dyrt.
  • Kun serverless. Kan ikke kjøre langlevd Python-prosess eller cron-jobber direkte. Bruk Railway eller Fly.io for det.
  • Overraskelsesregninger historisk. Flere kjente tilfeller har skjedd, selv om Vercel nå har spend management.
  • Hobby-lisens begrenset. Offisielt ikke-kommersiell på gratis-planen.
  • Vendor lock-in på Next.js-spesifikke funksjoner. ISR og Server Actions jobber best på Vercel.

Norsk bruk av Vercel

Vercels edge-nettverk har noder i Stockholm (ARN1) og Frankfurt (FRA1) som dekker Norge bra. Typisk TTFB fra Oslo er 20-40 ms for statiske sider. Ikke Cloudflare-nivå, men helt tilstrekkelig for de fleste applikasjoner.

GDPR: Vercel har standard DPA og SCC tilgjengelig. Datalagring primært i USA, men kan konfigureres til EU-only regions for Pro+. For offentlig sektor kan dette være en blocker — velg Hetzner eller DigitalOcean Frankfurt i stedet.

Når bør du velge Vercel

Vercel er riktig valg hvis:

  • Du bygger med Next.js eller React-baserte frameworks
  • Du vil ha beste mulig DX uten konfigurasjon
  • Du har budsjett for 200-1000 kr/mnd hosting
  • Du trenger preview deploys og team-samarbeid
  • Trafikk-volum er moderat (under 1 TB/mnd)

Vercel er ikke riktig hvis:

  • Du bygger WordPress-sider (bruk Kinsta eller SiteGround)
  • Du trenger langlevd backend (Python, Ruby — bruk Railway)
  • Budsjett er hovedprioritet (Cloudflare Pages eller Hetzner billigere)
  • Trafikken din er ekstrem (10+ TB/mnd — bruk Cloudflare)
  • Du har strenge datalokasjons-krav (bruk Hetzner EU-only)
